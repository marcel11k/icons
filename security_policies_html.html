Security and Data Handling Policies

Data Protection Overview

At our platform, we treat data security and privacy as fundamental. Our managed and self-hosted options combine enterprise-grade controls with developer-friendly APIs, ensuring customer data remains protected from ingestion through deletion.

Data Storage Security

Secure Database Infrastructure:

  • Data is stored in PostgreSQL databases with AES-256 encryption at rest
  • Row-level security policies enforce per-row access based on application-defined rules
  • Automatic daily backups with point-in-time recovery and configurable retention
  • Optional self-hosted deployment for full control over the physical environment

Physical Security Measures:

  • Hosting in tier-certified data centers with 24/7 monitoring, biometric access, and redundant systems
  • Geo-replication across selected regions for disaster recovery and continuity

Data Transmission Protection

  • TLS 1.3 enforced on all client-to-server and inter-service connections
  • JSON Web Tokens secure user sessions and API requests
  • Real-time streams and WebSockets operate over secure channels
  • Managed encryption key rotation via cloud key management services

Compliance Framework

Regulatory Adherence:

  • GDPR compliance for EU customers and CCPA coverage for California residents
  • SOC 2 Type II audited controls for managed offerings
  • HIPAA-eligible configurations available via self-host deployment

Data Processing Agreements:

  • Standard data processing agreement provided to all clients
  • Documentation of data flows across authentication, database, storage, and functions
  • Configurable data retention and automatic purging via policies and scheduled jobs

Access Control Policies

  • Native row-level security combined with service-role segregation at the API layer
  • Support for multi-factor authentication (TOTP, SMS) and external identity providers
  • Audit-ready logging via database extensions or external log integrations
  • Regular review and rotation of privileged keys and tokens

Incident Response

  • Published incident response playbook with roles, escalations, and SLAs
  • 24/7 monitoring of service health and anomaly detection alerts
  • Post-incident reports shared with affected customers
  • Continuous improvement based on post-mortem learnings

Regular Security Assessments

  • Quarterly vulnerability scans and annual penetration tests of infrastructure
  • Open code review and responsible disclosure policy for community contributions
  • Automated CI/CD checks, including SQL linting and dependency audits

Employee Security Practices

  • Mandatory security awareness and data handling training
  • Background and reference checks for staff with elevated privileges
  • Confidentiality and acceptable-use agreements enforced at hire

Client Control and Transparency

  • Web console and APIs provide visibility into schema, permissions, and usage metrics
  • Data export via database dump, storage API, or backup scripts
  • Self-serve tools and APIs for data subject access requests (DSAR)
  • Customizable retention schedules and on-demand record deletion

Third-Party Risk Management

  • Vetting process for extensions and third-party integrations before production use
  • Contractual security clauses for managed add-ons and partner services
  • Minimal necessary data sharing, controlled by row-level policies and service roles

Continuous Improvement

  • Weekly platform updates informed by emerging threat intelligence
  • Active participation in security communities (CVE, OWASP, etc.)
  • Quarterly newsletter on new security features and best practices
  • Annual policy review to maintain alignment with compliance requirements
